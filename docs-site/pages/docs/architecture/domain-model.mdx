The domain model defines the core business entities, their relationships, and state machines.

## Core Entities

### Intent

An intent represents a user's desire to swap tokens. It is locked in an on-chain escrow until a solver fills it or the user cancels.

```typescript
interface Intent {
  id: string;
  status: IntentStatus;
  creator: string;              // Wallet address (Bech32)
  inputPolicyId: string;        // "" for ADA
  inputAssetName: string;       // "" for ADA
  inputAmount: bigint;          // Lovelace/smallest unit
  outputPolicyId: string;
  outputAssetName: string;
  minOutput: bigint;            // Minimum acceptable output
  actualOutput?: bigint;        // Actual received (after fill)
  deadline: bigint;             // Slot number
  partialFill: boolean;
  maxPartialFills: number;
  fillCount: number;
  remainingInput: bigint;
  escrowTxHash?: string;
  escrowOutputIdx?: number;
  settlementTxHash?: string;
  solverAddress?: string;
}
```

### Pool

A constant-product AMM liquidity pool with two assets.

```typescript
interface Pool {
  id: string;
  poolNftPolicyId: string;      // Unique pool identifier NFT
  poolNftAssetName: string;
  assetA: AssetInfo;            // First asset (typically ADA)
  assetB: AssetInfo;            // Second asset
  reserveA: bigint;             // Current reserve of A
  reserveB: bigint;             // Current reserve of B
  totalLpTokens: bigint;        // LP tokens in circulation
  feeNumerator: number;         // Fee basis points (e.g., 30 = 0.30%)
  protocolFeeAccA: bigint;      // Accumulated protocol fees (A)
  protocolFeeAccB: bigint;      // Accumulated protocol fees (B)
  tvlAda: bigint;               // Total value locked
  state: PoolState;
  txHash: string;               // Current UTxO reference
  outputIndex: number;
  lpPolicyId?: string;          // LP token minting policy
}

interface AssetInfo {
  policyId: string;
  assetName: string;
  decimals: number;
  ticker?: string;
}
```

### Order

An advanced order (Limit, DCA, or Stop-Loss) locked in an on-chain escrow with execution conditions.

```typescript
interface Order {
  id: string;
  type: OrderType;              // LIMIT | DCA | STOP_LOSS
  status: OrderStatus;
  creator: string;
  inputPolicyId: string;
  inputAssetName: string;
  outputPolicyId: string;
  outputAssetName: string;
  // Limit / Stop-Loss fields
  inputAmount?: bigint;
  priceNumerator?: bigint;
  priceDenominator?: bigint;
  // DCA fields
  totalBudget?: bigint;
  amountPerInterval?: bigint;
  intervalSlots?: number;
  remainingBudget?: bigint;
  executedIntervals?: number;
  // Common
  deadline: bigint;
  escrowTxHash?: string;
  escrowOutputIdx?: number;
}
```

## Value Objects

Value objects are immutable and compared by value rather than identity.

### Asset Unit

```typescript
// Represents a Cardano native asset: policyId + assetName
// ADA is represented as empty strings: { policyId: "", assetName: "" }
type AssetUnit = {
  policyId: string;   // 56 hex chars (28 bytes) or "" for ADA
  assetName: string;  // Hex-encoded asset name or "" for ADA
};

// Helper: convert to Lucid-style unit string
function toUnit(asset: AssetUnit): string {
  if (!asset.policyId) return 'lovelace';
  return asset.policyId + asset.assetName;
}
```

### Amount

```typescript
// All amounts are BigInt to prevent floating-point errors
// ADA amounts are in lovelace (1 ADA = 1,000,000 lovelace)
type Amount = bigint;

// Display conversion
function toDisplay(amount: bigint, decimals: number): string {
  const str = amount.toString().padStart(decimals + 1, '0');
  const whole = str.slice(0, -decimals) || '0';
  const frac = str.slice(-decimals);
  return `${whole}.${frac}`;
}
```

## Enums

### IntentStatus

| Value | Description | Valid Transitions |
|-------|-------------|-------------------|
| `CREATED` | Initial state | → `PENDING` |
| `PENDING` | TX submitted to chain | → `ACTIVE` |
| `ACTIVE` | Escrow confirmed on-chain | → `FILLING`, `CANCELLING`, `EXPIRED` |
| `FILLING` | Solver building settlement | → `FILLED`, `PARTIALLY_FILLED`, `ACTIVE` (on failure) |
| `PARTIALLY_FILLED` | Some input consumed | → `FILLING`, `FILLED`, `EXPIRED` |
| `FILLED` | Fully settled | Terminal state |
| `CANCELLING` | Cancel TX submitted | → `CANCELLED` |
| `CANCELLED` | Cancel confirmed | Terminal state |
| `EXPIRED` | Deadline passed | → `RECLAIMED` |
| `RECLAIMED` | Expired funds recovered | Terminal state |

### OrderType

| Value | Trigger Condition | Use Case |
|-------|-------------------|----------|
| `LIMIT` | Price reaches target | Buy/sell at a specific price |
| `DCA` | Time interval elapsed | Dollar-cost average over time |
| `STOP_LOSS` | Price drops below threshold | Automatic risk management |

### PoolState

| Value | Description |
|-------|-------------|
| `ACTIVE` | Pool is live and accepting trades |
| `INACTIVE` | Pool is paused by admin |
| `CREATING` | Pool creation TX pending confirmation |

### CandleInterval

| Value | Duration | Use Case |
|-------|----------|----------|
| `M1` | 1 minute | Scalping / real-time |
| `M5` | 5 minutes | Short-term trading |
| `M15` | 15 minutes | Intraday analysis |
| `H1` | 1 hour | Swing trading |
| `H4` | 4 hours | Position trading |
| `D1` | 1 day | Daily analysis |
| `W1` | 1 week | Long-term trends |
