## Core Subsystems

| Subsystem | Technology | Role |
|-----------|-----------|------|
| **Frontend** | Next.js 16, React 19, Tailwind CSS | User interface & wallet integration |
| **Backend** | Express, TypeScript, Prisma | API server, business logic, TX building |
| **Solver Engine** | TypeScript (background service) | Monitors intents, routes through AMM, settles on-chain |
| **Smart Contracts** | Aiken (Plutus V3) | On-chain escrow, AMM pool, token minting |
| **Database** | PostgreSQL (Supabase) | State persistence, analytics, history |
| **Cache** | Upstash Redis | Quote caching, rate limiting, session data |
| **Chain Indexer** | Blockfrost API | On-chain data queries, TX submission |

## Design Principles

1. **Intent-Based Architecture** — Users express trade intentions; solvers compete to fulfill them optimally
2. **Clean Architecture** — Domain logic is independent of frameworks and infrastructure
3. **Hexagonal Ports & Adapters** — All external dependencies are behind interfaces (ports)
4. **Event-Driven** — WebSocket events propagate real-time state changes
5. **BigInt Arithmetic** — All financial computations use integer math to avoid floating-point errors
6. **On-Chain Settlement** — All trades settle through Plutus V3 validators for trustless execution
