# Clean Architecture

import { Callout } from 'nextra/components'

SolverNet DEX backend follows **Clean Architecture** principles, ensuring that business logic is independent of frameworks, databases, and external services.

## Layer Diagram

```
┌─────────────────────────────────────────────────┐
│              Interface Layer                      │
│  HTTP Routes, WebSocket Handlers, Middleware      │
│  Dependencies: Express, ws                        │
├─────────────────────────────────────────────────┤
│              Application Layer                    │
│  Use Cases, Application Services                  │
│  Dependencies: Domain only                        │
├─────────────────────────────────────────────────┤
│              Domain Layer (Core)                  │
│  Entities, Value Objects, Ports                   │
│  Dependencies: NONE                               │
├─────────────────────────────────────────────────┤
│              Infrastructure Layer                 │
│  Database, Chain, Cache, Cron                     │
│  Dependencies: Everything                         │
└─────────────────────────────────────────────────┘
```

## Dependency Rule

> Dependencies point **inward**. The Domain layer has zero external dependencies. The Application layer depends only on Domain. Infrastructure and Interface layers depend on everything above them.

## Domain Layer

The innermost layer contains pure business logic with no framework dependencies.

### Entities


### Ports (Interfaces)

Ports define **what** the domain needs from the outside world, without specifying **how**:

```typescript
// Ports are interfaces — the domain says WHAT it needs
interface IChainProvider {
  getUtxosAtAddress(address: string): Promise<UTxO[]>
  getChainTip(): Promise<ChainTip>
  submitTx(cbor: string): Promise<string>
  awaitTx(txHash: string): Promise<boolean>
}

interface IIntentRepository {
  findById(id: string): Promise<Intent | null>
  findActive(): Promise<Intent[]>
  findByUtxoRef(txHash: string, index: number): Promise<Intent | null>
  save(intent: Intent): Promise<void>
}

interface IPoolRepository {
  findById(id: string): Promise<Pool | null>
  findByAssets(assetA: string, assetB: string): Promise<Pool | null>
  findAll(options?: QueryOptions): Promise<Pool[]>
  save(pool: Pool): Promise<void>
}

interface ITxBuilder {
  buildEscrowTx(params: EscrowParams): Promise<UnsignedTx>
  buildSettlementTx(batch: IntentBatch): Promise<UnsignedTx>
  buildCancelTx(intent: Intent): Promise<UnsignedTx>
  // ... more operations
}
```

## Infrastructure Layer

Concrete implementations of domain ports:

```typescript
// backend/src/infrastructure/database/PrismaIntentRepository.ts
class PrismaIntentRepository implements IIntentRepository {
  constructor(private prisma: PrismaClient) {}

  async findActive(): Promise<Intent[]> {
    const records = await this.prisma.intent.findMany({
      where: { status: 'ACTIVE' }
    })
    return records.map(r => Intent.fromPersistence(r))
  }
}
```

## Benefits

| Benefit | How |
|---|---|
| **Testability** | Domain and application layers can be tested with mock implementations of ports |
| **Flexibility** | Swap Blockfrost for Ogmios/Kupo without touching business logic |
| **Clarity** | Each layer has a clear responsibility and dependency direction |
| **Maintainability** | Changes in one layer don't cascade to others |
