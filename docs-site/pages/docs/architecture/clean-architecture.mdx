# Clean Architecture

import { Callout } from 'nextra/components'

SolverNet DEX follows **Clean Architecture** (Hexagonal / Ports & Adapters) to ensure the domain logic is independent of frameworks, databases, and external services.

## Layer Diagram

```
┌─────────────────────────────────────────┐
│          Interface Layer                 │
│  Express Routes, WebSocket, Middleware   │
│  (Depends on Application Layer)         │
├─────────────────────────────────────────┤
│         Application Layer                │
│  Use Cases, Services, DTOs              │
│  (Depends on Domain Layer)              │
├─────────────────────────────────────────┤
│           Domain Layer                   │
│  Entities, Value Objects, Ports, Errors  │
│  (No external dependencies)             │
├─────────────────────────────────────────┤
│       Infrastructure Layer               │
│  Prisma, Blockfrost, Lucid, Redis, Cron │
│  (Implements Domain Ports)              │
└─────────────────────────────────────────┘
```

## Dependency Rule

<Callout type="warning">
Dependencies flow **inward only**. Inner layers NEVER import from outer layers. The Domain layer has ZERO external dependencies.
</Callout>

```
Interface → Application → Domain ← Infrastructure
```

- **Interface** imports Application (use cases)
- **Application** imports Domain (entities, ports)
- **Infrastructure** implements Domain ports
- **Domain** imports nothing external

## Domain Layer

The innermost layer contains pure business logic with no framework dependencies.

### Entities

```typescript
// domain/entities/intent.ts
export interface Intent {
  id: string;
  status: IntentStatus;
  creator: string;
  inputPolicyId: string;
  inputAssetName: string;
  inputAmount: bigint;
  outputPolicyId: string;
  outputAssetName: string;
  minOutput: bigint;
  deadline: bigint;
  partialFill: boolean;
  // ...
}
```

### Ports (Interfaces)

```typescript
// domain/ports/intent-repository.ts
export interface IntentRepository {
  findById(id: string): Promise<Intent | null>;
  findActive(): Promise<Intent[]>;
  findByCreator(address: string, options?: PaginationOptions): Promise<PaginatedResult<Intent>>;
  create(intent: CreateIntentData): Promise<Intent>;
  updateStatus(id: string, status: IntentStatus): Promise<Intent>;

```

### Errors

```typescript

export class DomainError extends Error {
  constructor(
    public readonly code: string,
    message: string,
    public readonly httpStatus: number = 400

    super(message);
  }
}
```

## Infrastructure Layer

Implements domain ports with concrete technologies.

```typescript
// infrastructure/database/prisma-intent-repository.ts
export class PrismaIntentRepository implements IntentRepository {
  constructor(private prisma: PrismaClient) {}

  async findById(id: string): Promise<Intent | null> {
    const record = await this.prisma.intent.findUnique({ where: { id } });
    return record ? this.toDomain(record) : null;
  }

  async findActive(): Promise<Intent[]> {
    const records = await this.prisma.intent.findMany({
      where: { status: 'ACTIVE' }
    });
    return records.map(this.toDomain);
  }
}
```

## Interface Layer

Maps HTTP requests to use case invocations.

```typescript
// interface/http/routes/intent-routes.ts
router.post('/intents', validate(intentSchema), async (req, res, next) => {
  try {
    const result = await createIntentUseCase.execute(req.body);
    res.status(201).json(result);
  } catch (error) {
    next(error);
  }
});
```

## Benefits

| Principle | Benefit |
|-----------|---------|
| **Testability** | Domain logic can be unit tested without infrastructure |
| **Flexibility** | Swap PostgreSQL for MongoDB by implementing the same ports |
| **Maintainability** | Changes to one layer don't cascade to others |
| **Scalability** | Each layer can be scaled independently |
| **Onboarding** | Clear boundaries make the codebase navigable |
