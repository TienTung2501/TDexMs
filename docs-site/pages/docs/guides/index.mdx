## How SolverNet Works — Quick Recap

```
User creates Intent → Solver collects batch → Netting offsets opposing flows
  → AMM fills residual → Users receive tokens on-chain
```

| Concept | Description |
|---------|-------------|
| **Intent** | A signed request to swap token A for token B, locked in an escrow UTxO |
| **Solver** | An off-chain bot that collects intents, optimizes via netting, and builds settlement TXs |
| **Netting** | Matching opposing swaps directly (e.g., ADA→MIN vs MIN→ADA) to reduce AMM usage |
| **AMM Pool** | On-chain constant-product liquidity pool that fills any residual after netting |
| **Escrow** | Smart contract holding user funds until settlement, cancellation, or reclaim |

## Typical Flow

1. **Connect wallet** — CIP-30 compatible wallet
2. **Select tokens** — Choose input and output token
3. **Get quote** — Backend calculates expected output, fees, and price impact
4. **Create intent** — Backend builds TX, user signs, TX submitted on-chain
5. **Wait for settlement** — Solver processes within ~2 minutes
6. **Receive tokens** — Output sent directly to your wallet address
