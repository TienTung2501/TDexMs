# Advanced Orders

import { Callout, Tabs } from 'nextra/components'

SolverNet DEX supports three advanced order types beyond simple market swaps.

## Order Types

| Type | Description | Use Case |
|---|---|---|
| **Limit** | Execute when price reaches target | "Buy TOKEN at 0.45 ADA" |
| **DCA** | Execute fixed amount at regular intervals | "Buy 100 ADA of TOKEN every hour" |
| **Stop-Loss** | Execute when price drops below threshold | "Sell TOKEN if price drops below 0.40 ADA" |

## Limit Orders

A limit order executes when the market price reaches your specified target price.

**How it works:**
1. You lock funds in an order escrow UTxO
2. The order executor cron checks prices every 60 seconds
3. When market price meets your target, the order is executed
4. Output tokens are delivered to your wallet

**Price condition:**
$$
\frac{\text{outputAmount}}{\text{inputConsumed}} \geq \frac{\text{priceNumerator}}{\text{priceDenominator}}
$$

**Example — Buy TOKEN at 0.45 ADA:**
```json
{
  "orderType": "LIMIT",
  "inputAsset": "lovelace",
  "outputAsset": "abc123.TOKEN",
  "totalBudget": "1000000000",
  "priceNumerator": "2222222",
  "priceDenominator": "1000000",
  "expiresInSeconds": 86400
}
```

This means: I want at least 2.222 TOKEN per ADA (equivalent to buying at 0.45 ADA/TOKEN).

## DCA (Dollar-Cost Averaging)

DCA orders execute a fixed amount at regular intervals, spreading your entry across time to reduce timing risk.

**How it works:**
1. You lock a total budget in an order escrow
2. Every `dcaInterval` seconds, the executor swaps `dcaAmountPerInterval`
3. Partial fills create a new escrow with reduced budget
4. Continues until budget is exhausted or order expires

**Example — Buy TOKEN with 10,000 ADA over 10 hours:**
```json
{
  "orderType": "DCA",
  "inputAsset": "lovelace",
  "outputAsset": "abc123.TOKEN",
  "totalBudget": "10000000000",
  "dcaInterval": 3600,
  "dcaAmountPerInterval": "1000000000",
  "expiresInSeconds": 604800
}
```

This executes 1,000 ADA worth of TOKEN every hour for 10 hours.

<Callout type="info">
DCA orders don't have a target price by default — they execute at whatever the current market price is. You can optionally set a `priceCap` to skip execution when prices are too high.
</Callout>

## Stop-Loss Orders

Stop-loss orders automatically sell your position when the price drops below a threshold, limiting your downside.

**How it works:**
1. You lock your tokens in an order escrow
2. The executor monitors the current price
3. When price drops below your stop-loss price, the order executes
4. Output (usually ADA) is delivered to your wallet

**Example — Sell TOKEN if price drops below 0.40 ADA:**
```json
{
  "orderType": "STOP_LOSS",
  "inputAsset": "abc123.TOKEN",
  "outputAsset": "lovelace",
  "totalBudget": "5000000000",
  "priceNumerator": "400000",
  "priceDenominator": "1000000",
  "expiresInSeconds": 604800
}
```

## Order Lifecycle

```
Created → Pending → Active → [Partially Filled →] Filled
                      ↓
                  Cancelled
                      ↓
                   Expired
```

| State | Description |
|---|---|
| `CREATED` | Order record created in DB |
| `PENDING` | Escrow TX submitted, awaiting confirmation |
| `ACTIVE` | On-chain, being monitored by executor |
| `PARTIALLY_FILLED` | DCA order with some executions completed |
| `FILLED` | Fully executed |
| `CANCELLED` | User cancelled the order |
| `EXPIRED` | TTL expired, funds reclaimable |

## Managing Orders

### Via UI

Navigate to `/orders` to:
- View all active orders
- See execution history
- Cancel pending orders
- Track DCA progress

### Via API

```bash
# List your orders
curl "http://localhost:3001/v1/orders?creator=addr_test1qz..."

# Cancel an order
curl -X DELETE "http://localhost:3001/v1/orders/order_abc123" \
  -H "Content-Type: application/json" \
  -d '{"creatorAddress": "addr_test1qz..."}'
```

## Security

All order types share the same security properties as intents:

- **Non-custodial** — Funds locked in a validator, not held by anyone
- **Cancellable** — You can always cancel and get your funds back
- **Minimum output** — On-chain enforcement prevents bad execution
- **Expiry** — Orders automatically expire and funds are reclaimable
