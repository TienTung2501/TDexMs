# Configuration

import { Callout } from 'nextra/components'

Complete environment variable reference for the SolverNet DEX backend.

## Environment File

Create a `.env` file in the `backend/` directory:

```bash
cd backend
cp .env.example .env
```

---

## Required Variables

| Variable | Type | Description |
|----------|------|-------------|
| `DATABASE_URL` | string | PostgreSQL connection string |
| `BLOCKFROST_PROJECT_ID` | string | Blockfrost API key for Cardano access |

## Server

| Variable | Default | Description |
|----------|---------|-------------|
| `NODE_ENV` | `development` | Runtime environment (`development`, `production`, `test`) |
| `PORT` | `3001` | HTTP server port |
| `HOST` | `0.0.0.0` | Bind address |
| `LOG_LEVEL` | `info` | Logging level (`fatal`, `error`, `warn`, `info`, `debug`, `trace`) |
| `CORS_ORIGIN` | `http://localhost:3000` | Allowed CORS origins (comma-separated for multiple) |

## Database

| Variable | Default | Description |
|----------|---------|-------------|
| `DATABASE_URL` | **required** | PostgreSQL connection string (Supabase or local) |

## Cache (Optional)

| Variable | Default | Description |
|----------|---------|-------------|
| `UPSTASH_REDIS_URL` | `''` | Upstash Redis REST URL |
| `UPSTASH_REDIS_TOKEN` | `''` | Upstash Redis auth token |

<Callout type="info">
Redis is optional. Without it, the system falls back to database queries with slightly higher latency.
</Callout>

## Cardano Network

| Variable | Default | Description |
|----------|---------|-------------|
| `CARDANO_NETWORK` | `preprod` | Target network (`preprod`, `preview`, `mainnet`) |
| `BLOCKFROST_URL` | `https://cardano-preprod.blockfrost.io/api/v0` | Blockfrost API base URL |
| `BLOCKFROST_PROJECT_ID` | `''` | Blockfrost project API key |

## Smart Contract Addresses

| Variable | Default | Description |
|----------|---------|-------------|
| `ESCROW_SCRIPT_ADDRESS` | `''` | Escrow validator Bech32 address |
| `POOL_SCRIPT_ADDRESS` | `''` | Pool validator Bech32 address |
| `SETTINGS_NFT_POLICY_ID` | `''` | Settings NFT minting policy ID |
| `SETTINGS_NFT_ASSET_NAME` | `''` | Settings NFT asset name |
| `ADMIN_ADDRESS` | `''` | Factory admin wallet address (Bech32) |

<Callout type="warning">
These addresses are derived from the compiled Aiken smart contracts. Use the Admin panel's "Deploy Settings" and "Deploy Factory" to set them up automatically.
</Callout>

## Solver Engine

| Variable | Default | Description |
|----------|---------|-------------|
| `SOLVER_ENABLED` | `false` | Enable the solver engine background service |
| `SOLVER_SEED_PHRASE` | `''` | Solver wallet mnemonic (24 words) |
| `SOLVER_ADDRESS` | `''` | Solver wallet Bech32 address |
| `SOLVER_BATCH_WINDOW_MS` | `5000` | Batch collection window (milliseconds) |
| `SOLVER_MAX_RETRIES` | `3` | Maximum TX submission retries |
| `SOLVER_MIN_PROFIT_LOVELACE` | `100000` | Minimum profit threshold (lovelace) |

<Callout type="error">
**Security:** Never commit `SOLVER_SEED_PHRASE` to version control. Use environment variables or secrets management.
</Callout>

## Order System

| Variable | Default | Description |
|----------|---------|-------------|
| `ORDER_ROUTES_ENABLED` | `false` | Enable order API routes (Limit/DCA/StopLoss) |
| `ORDER_EXECUTOR_ENABLED` | `false` | Enable order execution cron job |

## Rate Limiting

| Variable | Default | Description |
|----------|---------|-------------|
| `RATE_LIMIT_WINDOW_MS` | `60000` | Rate limit window (1 minute) |
| `RATE_LIMIT_MAX` | `100` | Max requests per window (read endpoints) |

Write endpoints (POST/DELETE) are limited to `RATE_LIMIT_MAX / 5` per window.

## Authentication

| Variable | Default | Description |
|----------|---------|-------------|
| `JWT_SECRET` | `dev-secret-change-in-production` | JWT signing secret |
| `JWT_EXPIRES_IN` | `24h` | JWT token TTL |

## Chart & Analytics

| Variable | Default | Description |
|----------|---------|-------------|
| `CHART_SNAPSHOT_INTERVAL_MS` | `60000` | Price aggregation cron interval |
| `CHART_MAX_CANDLES` | `500` | Max candles returned per query |
| `CHAIN_SYNC_INTERVAL_MS` | `120000` | Pool chain sync polling interval |

## Faucet (Testnet)

| Variable | Default | Description |
|----------|---------|-------------|
| `FAUCET_TARGET_ADDRESS` | `''` | Testnet faucet target address |
| `FAUCET_API_KEY` | `''` | Faucet API key |
| `FAUCET_INTERVAL_MS` | `86400000` | Faucet request interval (24h) |

## Example `.env` Files

### Development (Preprod Testnet)

```env
NODE_ENV=development
PORT=3001
HOST=0.0.0.0
LOG_LEVEL=debug

DATABASE_URL="postgresql://postgres:password@db.xxx.supabase.co:5432/postgres"
UPSTASH_REDIS_URL=""
UPSTASH_REDIS_TOKEN=""

CARDANO_NETWORK=preprod
BLOCKFROST_URL="https://cardano-preprod.blockfrost.io/api/v0"
BLOCKFROST_PROJECT_ID="preprodXXXXXXXXXXXXX"

ESCROW_SCRIPT_ADDRESS=""
POOL_SCRIPT_ADDRESS=""
SETTINGS_NFT_POLICY_ID=""
SETTINGS_NFT_ASSET_NAME=""
ADMIN_ADDRESS="addr_test1qz..."

SOLVER_ENABLED=false
SOLVER_SEED_PHRASE=""
SOLVER_ADDRESS=""

ORDER_ROUTES_ENABLED=false
ORDER_EXECUTOR_ENABLED=false

CORS_ORIGIN="http://localhost:3000"
JWT_SECRET="dev-secret-change-in-production"
```

### Production (Mainnet)

```env
NODE_ENV=production
PORT=3001
HOST=0.0.0.0
LOG_LEVEL=warn

DATABASE_URL="postgresql://..."
UPSTASH_REDIS_URL="https://xxxxx.upstash.io"
UPSTASH_REDIS_TOKEN="AXxxxx..."

CARDANO_NETWORK=mainnet
BLOCKFROST_URL="https://cardano-mainnet.blockfrost.io/api/v0"
BLOCKFROST_PROJECT_ID="mainnetXXXXXXXXXXXXX"

ESCROW_SCRIPT_ADDRESS="addr1w..."
POOL_SCRIPT_ADDRESS="addr1w..."
SETTINGS_NFT_POLICY_ID="..."
SETTINGS_NFT_ASSET_NAME="..."
ADMIN_ADDRESS="addr1q..."

SOLVER_ENABLED=true
SOLVER_SEED_PHRASE="word1 word2 ... word24"
SOLVER_ADDRESS="addr1q..."
SOLVER_BATCH_WINDOW_MS=10000
SOLVER_MAX_RETRIES=5
SOLVER_MIN_PROFIT_LOVELACE=500000

ORDER_ROUTES_ENABLED=true
ORDER_EXECUTOR_ENABLED=true

CORS_ORIGIN="https://yourdomain.com"
JWT_SECRET="your-production-secret-min-32-chars"

RATE_LIMIT_MAX=50
```
