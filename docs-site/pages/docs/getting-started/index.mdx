## What is SolverNet DEX?

SolverNet DEX is a next-generation DEX that replaces the traditional "user vs pool" swap model with an **intent-based architecture**:

1. **Users express intents** — "I want to swap X amount of Token A for at least Y amount of Token B"
2. **Intents are locked on-chain** — Each intent is a separate UTxO at the escrow validator address
3. **Solvers aggregate and settle** — Off-chain solver bots collect intents, cross-match opposing trades (netting), and settle them in optimized batched transactions

This design eliminates eUTxO concurrency issues, reduces price impact through netting, and provides MEV protection.

## Key Concepts

| Concept | Description |
|---|---|
| **Intent** | A signed swap request locked in an escrow UTxO. Contains: input asset, output asset, minimum output, owner address, expiry |
| **Solver** | An off-chain bot that collects intents, finds optimal routes, and submits settlement transactions |
| **Netting** | Cross-matching opposing intents (A→B vs B→A) to reduce the amount that needs to go through the AMM pool |
| **Escrow** | The on-chain validator that holds user funds until settlement or cancellation |
| **Pool** | A constant-product AMM pool (x·y=k) that handles the residual after netting |
| **Settlement** | A batched transaction that consumes multiple escrow UTxOs, interacts with the pool, and delivers outputs |
