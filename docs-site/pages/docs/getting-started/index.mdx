## What is SolverNet DEX?

SolverNet DEX introduces an **intent-based trading model** to Cardano:

1. **Users submit trade intentions** (intents) instead of executing swaps directly
2. **A solver engine** monitors these intents and finds optimal execution routes through AMM liquidity pools
3. **On-chain settlement** happens through Plutus V3 validators, ensuring trustless, verifiable execution

This architecture enables features like **trade netting** (matching opposing trades), **multi-hop routing**, and **advanced order types** (Limit, DCA, Stop-Loss) — all settled on-chain.

## Key Concepts

| Concept | Description |
|---------|-------------|
| **Intent** | A user's declared desire to swap tokens, locked in an on-chain escrow |
| **Solver** | An automated engine that matches intents and executes optimal trades |
| **Pool** | A constant-product AMM liquidity pool (x * y = k) |
| **Escrow** | On-chain UTxO holding user's tokens until the intent is filled or cancelled |
| **LP Token** | Proportional ownership token for liquidity providers |
| **Settlement** | On-chain transaction that fulfills an intent and transfers tokens |

## Tech Stack

| Layer | Technology | Purpose |
|-------|-----------|---------|
| **Frontend** | Next.js 16, React 19, Tailwind CSS | Trading interface |
| **Backend** | Express, TypeScript, Prisma | API & business logic |
| **Smart Contracts** | Aiken (Plutus V3) | On-chain validators |
| **Database** | PostgreSQL (Supabase) | State persistence |
| **Cache** | Upstash Redis | Performance optimization |
| **Chain Access** | Blockfrost API | Cardano data + TX submission |
| **TX Builder** | @lucid-evolution/lucid | Cardano TX construction |
| **Wallet** | CIP-30 (Nami, Eternl, Lace) | User authentication & signing |
