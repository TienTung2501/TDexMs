# Datum & Redeemer Reference

import { Callout } from 'nextra/components'

This page provides a complete reference for all on-chain datum and redeemer types used across SolverNet DEX smart contracts.

## Type Definitions (Aiken)

### Common Types

```aiken
type AssetClass {
  policy_id: PolicyId,
  asset_name: AssetName,
}

type Address {
  payment_credential: Credential,
  stake_credential: Option<Credential>,
}

type POSIXTime = Int  // Milliseconds since Unix epoch
```

## Escrow Validator

### Datum

```aiken
type EscrowDatum {
  /// Wallet address that created this intent
  owner: Address,
  /// Asset being sold by the user
  input_asset: AssetClass,
  /// Asset desired by the user
  output_asset: AssetClass,
  /// Minimum amount of output_asset the user will accept
  min_output: Int,
  /// One-shot authentication token
  intent_token: AssetClass,
  /// Unix timestamp (ms) after which the intent expires
  expiry_time: POSIXTime,
}
```

### Redeemers

```aiken
type EscrowRedeemer {
  /// Owner cancels their intent, funds returned
  Cancel
  /// Solver fills the intent (full or partial)
  Fill { 
    /// Amount of output_asset delivered
    output_amount: Int 
  }
  /// Anyone reclaims expired intent funds to the owner
  Reclaim
}
```

## Pool Validator

### Datum

```aiken
type PoolDatum {
  /// First asset in the pair (canonical: A < B)
  asset_a: AssetClass,
  /// Second asset in the pair
  asset_b: AssetClass,
  /// Total LP tokens ever minted (minus burned)
  total_lp_tokens: Int,
  /// Swap fee numerator (e.g., 30)
  fee_numerator: Int,
  /// Swap fee denominator (always 10000)
  fee_denominator: Int,
  /// Accumulated protocol fees in asset A
  protocol_fee_a: Int,
  /// Accumulated protocol fees in asset B
  protocol_fee_b: Int,
  /// Pool identity NFT
  pool_nft: AssetClass,
}
```

### Redeemers

```aiken
type PoolRedeemer {
  /// Swap asset A for B or B for A
  Swap {
    direction: SwapDirection,
    input_amount: Int,
    min_output: Int,
  }
  /// Add liquidity to the pool
  Deposit {
    amount_a: Int,
    amount_b: Int,
    min_lp: Int,
  }
  /// Remove liquidity from the pool
  Withdraw {
    lp_amount: Int,
    min_a: Int,
    min_b: Int,
  }
  /// Admin collects accumulated protocol fees
  CollectFees
  /// Admin closes the pool
  ClosePool
}

type SwapDirection {
  AToB
  BToA
}
```

## Order Validator

### Datum

```aiken
type OrderDatum {
  /// Wallet address that created the order
  creator: Address,
  /// Order type
  order_type: OrderType,
  /// Asset being sold
  input_asset: AssetClass,
  /// Asset being bought
  output_asset: AssetClass,
  /// Total budget allocated to this order
  total_budget: Int,
  /// Budget remaining (decreases with partial executions)
  remaining_budget: Int,
  /// Target price (numerator / denominator)
  price_numerator: Int,
  price_denominator: Int,
  /// DCA: interval between executions (ms)
  dca_interval: Option<Int>,
  /// DCA: amount to swap per interval
  dca_amount: Option<Int>,
  /// DCA: timestamp of last execution
  last_executed: Option<POSIXTime>,
  /// Authentication token
  intent_token: AssetClass,
  /// Expiry timestamp
  expiry_time: POSIXTime,
}

type OrderType {
  Limit
  DCA
  StopLoss
}
```

### Redeemers

```aiken
type OrderRedeemer {
  /// Creator cancels the order
  CancelOrder
  /// Solver executes (partially or fully) 
  ExecuteOrder {
    input_consumed: Int,
    output_delivered: Int,
  }
  /// Anyone reclaims expired order
  ReclaimOrder
}
```

## Factory Validator

### Datum

```aiken
type FactoryDatum {
  /// List of registered pool identities

type PoolEntry {
```

```aiken
type FactoryRedeemer {
```

## Settings Validator

### Datum

```aiken
type SettingsDatum {

  version: Int,
```

### Redeemers

```aiken
type SettingsRedeemer {
  /// Update settings (admin only)
  UpdateSettings
}
```

## Serialization

All datums use **inline datums** (Plutus V3 feature) — the datum is embedded directly in the UTxO output, not referenced by hash. This means:

- No datum witness needed in transactions
- Datums are immediately readable by anyone querying the UTxO
- ChainSync service can parse pool/escrow state directly from Blockfrost

<Callout type="info">
The backend uses Lucid Evolution's `Data.to()` and `Data.from()` for serialization between TypeScript objects and on-chain CBOR-encoded Plutus Data.
</Callout>
