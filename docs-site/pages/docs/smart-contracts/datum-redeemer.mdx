# Datum & Redeemer Reference

import { Callout } from 'nextra/components'

Complete type reference for all on-chain data structures defined in `lib/solvernet/types.ak`.

## Asset Types

```aiken
/// Represents a native Cardano asset (policy ID + asset name)
type AssetClass {
  policy_id: PolicyId,       // 28-byte policy ID ("" for ADA)
  asset_name: AssetName,     // Asset name bytes ("" for ADA)
}
```

ADA is represented as `AssetClass { policy_id: "", asset_name: "" }`.

## Escrow Types

### EscrowDatum

```aiken
type EscrowDatum {
  creator: VerificationKeyHash,     // 28 bytes — owner pubkey hash
  input_policy_id: PolicyId,
  input_asset_name: AssetName,
  output_policy_id: PolicyId,
  output_asset_name: AssetName,
  min_output: Int,                  // Minimum acceptable output amount
  deadline: POSIXTime,              // UNIX timestamp in milliseconds
  escrow_token: AssetName,          // Intent token asset name (blake2b hash)
  input_amount: Int,                // Original input amount
  max_partial_fills: Int,           // Maximum partial fills allowed
  fill_count: Int,                  // Number of fills completed
  remaining_input: Int,             // Input remaining to be filled
}
```

### EscrowRedeemer

```aiken
type EscrowRedeemer {
  Cancel                             // Owner cancels (before fill)
  Fill                               // Solver fills (fully or partially)
  Reclaim                            // Owner reclaims (after deadline)
}
```

### CBOR Encoding

```
Datum:  Constr(0, [creator, in_pid, in_an, out_pid, out_an,
                   min_output, deadline, escrow_token,
                   input_amount, max_partial_fills,
                   fill_count, remaining_input])

Cancel:  Constr(0, [])
Fill:    Constr(1, [])
Reclaim: Constr(2, [])
```

## Pool Types

### PoolDatum

```aiken
type PoolDatum {
  asset_a: AssetClass,              // First token (canonically ordered)
  asset_b: AssetClass,              // Second token
  reserve_a: Int,                   // Active reserve of asset A
  reserve_b: Int,                   // Active reserve of asset B
  total_lp_tokens: Int,             // Total LP tokens outstanding
  fee_numerator: Int,               // Fee rate (e.g., 30 = 0.30%)
  protocol_fee_a: Int,              // Accumulated protocol fees (asset A)
  protocol_fee_b: Int,              // Accumulated protocol fees (asset B)
  pool_nft: AssetClass,             // Thread token identity
  last_root_k: Int,                 // √(reserveA × reserveB) before swap
}
```

### PoolRedeemer

```aiken
type PoolRedeemer {
  Swap { direction: SwapDirection }  // Execute swap
  Deposit                            // Add liquidity
  Withdraw                           // Remove liquidity
  CollectFees                        // Admin collects protocol fees
  ClosePool                          // Admin closes pool
}

type SwapDirection {
  AToB                               // Swap asset A for asset B
  BToA                               // Swap asset B for asset A
}
```

### CBOR Encoding

```
Datum:  Constr(0, [asset_a, asset_b, reserve_a, reserve_b,
                   total_lp, fee_num, proto_fee_a, proto_fee_b,
                   pool_nft, last_root_k])

Swap:           Constr(0, [direction])
Deposit:        Constr(1, [])
Withdraw:       Constr(2, [])
CollectFees:    Constr(3, [])
ClosePool:      Constr(4, [])

SwapDirection:
  AToB: Constr(0, [])
  BToA: Constr(1, [])
```

## Order Types

### OrderDatum

```aiken
type OrderDatum {
  creator: VerificationKeyHash,
  input_asset: AssetClass,
  output_asset: AssetClass,
  budget: Int,                      // Total input budget
  min_output: Int,                  // Minimum total output
  deadline: POSIXTime,
  order_type: OrderType,
  order_params: OrderParams,
  filled_amount: Int,               // Total input filled so far
  fill_count: Int,
  escrow_token: AssetName,          // Intent token (thread)
}

type OrderType {
  Limit                              // Constr(0, [])
  DCA                                // Constr(1, [])
  StopLoss                           // Constr(2, [])
}

type OrderParams {
  amount_per_interval: Int,          // DCA: amount per execution
  interval: Int,                     // DCA: seconds between fills
  last_fill_time: Int,               // DCA: timestamp of last fill
}
```

### OrderRedeemer

```aiken
type OrderRedeemer {
  CancelOrder                        // Creator cancels
  ExecuteOrder                       // Solver executes
  ReclaimOrder                       // Creator reclaims after deadline
}
```

### CBOR Encoding

```
Datum:  Constr(0, [creator, input_asset, output_asset, budget,
                   min_output, deadline, order_type, order_params,
                   filled_amount, fill_count, escrow_token])

CancelOrder:    Constr(0, [])
ExecuteOrder:   Constr(1, [])
ReclaimOrder:   Constr(2, [])
```

## Serialization Notes

<Callout type="info">
All types use **CBOR** serialization via Plutus Data encoding. The `Constr(index, fields)` encoding maps Aiken constructors to Plutus Data.
</Callout>

### Constructor Index Mapping

Each variant in a sum type gets an incrementing constructor index:

```
type MyRedeemer {
  First    → Constr(0, [])
  Second   → Constr(1, [])
  Third    → Constr(2, [])
}
```

### Integer Encoding

All integers are arbitrary-precision `Integer` in Plutus. The backend uses JavaScript `BigInt` for all on-chain calculations to prevent overflow.

### ByteString Encoding

- `PolicyId` → 28-byte hex string (empty for ADA)
- `AssetName` → variable-length hex string (empty for ADA lovelace)
- `VerificationKeyHash` → 28-byte hex string
- `ScriptHash` → 28-byte hex string

### Parsing in Backend

The `IntentCollector` parses on-chain datums:

```typescript
// Example: parsing EscrowDatum from Constr(0, [...])
const fields = datum.fields; // Array of 12 Plutus Data fields
const escrowDatum = {
  creator: fields[0],              // ByteString
  inputPolicyId: fields[1],        // ByteString
  inputAssetName: fields[2],       // ByteString
  outputPolicyId: fields[3],       // ByteString
  outputAssetName: fields[4],      // ByteString
  minOutput: BigInt(fields[5]),     // Integer
  deadline: BigInt(fields[6]),      // Integer
  escrowToken: fields[7],          // ByteString
  inputAmount: BigInt(fields[8]),   // Integer
  maxPartialFills: Number(fields[9]),
  fillCount: Number(fields[10]),
  remainingInput: BigInt(fields[11]),
};
```
