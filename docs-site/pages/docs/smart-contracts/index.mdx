# Smart Contracts

import { Cards, Card, Callout } from 'nextra/components'

SolverNet's on-chain logic is written in **Aiken** (Plutus V3) and compiled to UPLC for Cardano.

## Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                    Smart Contract Suite                       │
│                                                              │
│  Validators (5)                  Minting Policies (3)        │
│  ┌──────────────────┐           ┌──────────────────────┐    │
│  │ Escrow Validator  │           │ Intent Token Policy   │   │
│  │ (swap intents)    │           │ (one-shot, no params) │   │
│  ├──────────────────┤           ├──────────────────────┤    │
│  │ Pool Validator    │           │ Pool NFT Policy       │   │
│  │ (AMM logic)       │           │ (thread token)        │   │
│  ├──────────────────┤           ├──────────────────────┤    │
│  │ Order Validator   │           │ LP Token Policy       │   │
│  │ (limit/DCA/stop)  │           │ (forwarding mint)     │   │
│  ├──────────────────┤           └──────────────────────┘    │
│  │ Factory Validator │                                       │
│  │ (pool creation)   │                                       │
│  ├──────────────────┤                                       │
│  │ Settings Validator│                                       │
│  │ (protocol config) │                                       │
│  └──────────────────┘                                       │
└─────────────────────────────────────────────────────────────┘
```

## Key Design Principles

| Principle | Implementation |
|-----------|---------------|
| **Non-custodial** | Users retain ownership via escrow; only creator can cancel |
| **Parameterized scripts** | Scripts reference each other via DAG for composability |
| **Thread tokens** | Pool NFT ensures UTxO continuity across state transitions |
| **One-shot tokens** | Intent tokens prevent replay and enable batch burns |
| **Anti-double-satisfaction** | InlineDatum with token name prevents one output serving two intents |
| **Integer arithmetic** | All math uses BigInt — no floating point on-chain |

## Security Properties

| Property | How Enforced |
|----------|-------------|
| Funds safety | Escrow validator requires creator signature for cancel |
| Price protection | `minOutput` check on every fill |
| Deadline enforcement | Validity intervals checked on-chain |
| Pool integrity | Constant product invariant + root K tracking |
| Admin authority | Script-hash-based admin via withdrawal check |
| LP token accuracy | Forwarding mint delegates to pool/factory validator |

## Constants

From `lib/solvernet/constants.ak`:

| Constant | Value | Description |
|----------|-------|-------------|
| `fee_denominator` | 10,000 | Fee precision base |
| `min_fee_numerator` | 1 | Minimum 0.01% fee |
| `max_fee_numerator` | 300 | Maximum 3% fee |
| `protocol_fee_share` | 6 | 1/6 of trading fee goes to protocol |
| `minimum_liquidity` | 1,000 | Locked LP tokens on pool creation |
| `min_utxo_lovelace` | 1,500,000 | Minimum ADA in UTxOs |
| `default_max_partial_fills` | 5 | Max partial fills per intent |
| `min_fill_percent` | 10 | Minimum 10% fill threshold |
| `min_dca_interval` | 180 | Minimum 3 minutes between DCA fills |
