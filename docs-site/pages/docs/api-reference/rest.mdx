# REST API Reference

import { Callout, Tabs } from 'nextra/components'

Complete reference for all REST API endpoints. All routes are prefixed with `/v1`.

<Callout type="info">
**Rate Limiting:** All `/v1` routes are subject to rate limiting (`RATE_LIMIT_MAX` requests per 60s window).
Write endpoints (POST/DELETE) use a stricter limit of `RATE_LIMIT_MAX / 5` per window.
</Callout>

---

## Health

### `GET /v1/health`

System health check with service status and Blockfrost usage.

**Response:**
```json
{
  "status": "healthy",
  "version": "1.0.0",
  "uptime": 3600,
  "services": {
    "database": "connected",
    "blockfrost": "connected",
    "cache": "connected"
  },
  "blockfrostUsage": {
    "callsToday": 1250,
    "dailyBudget": 50000,
    "remaining": 48750,
    "percentUsed": 2.5
  }
}
```

### `GET /v1/health/ready`

Kubernetes-style readiness probe. Checks database connectivity.

**Response (200):**
```json
{ "status": "ready" }
```

**Response (503):**
```json
{ "status": "not ready" }
```

## Quote

### `GET /v1/quote`

Calculate a swap quote with routing and price impact.

**Query Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `inputAsset` | string | Yes | Input asset unit (e.g., `lovelace`) |
| `outputAsset` | string | Yes | Output asset unit (policyId + assetName) |
| `inputAmount` | string | Conditional | Amount to swap (provide either input or output) |
| `outputAmount` | string | Conditional | Desired output amount |
| `slippage` | number | No | Slippage tolerance (default: 0.5%) |

**Response:**
```json
{
  "inputAsset": "lovelace",
  "outputAsset": "abc123def456.TOKEN",
  "inputAmount": "100000000",
  "outputAmount": "49251234",
  "minOutput": "48996478",
  "priceImpact": "0.50",
  "route": [
    {
      "poolId": "pool_abc123",
      "type": "direct",
      "inputAsset": "lovelace",
      "outputAsset": "abc123def456.TOKEN",
      "inputAmount": "100000000",
      "outputAmount": "49251234",
      "fee": "300000"
    }
  ],
  "estimatedFees": {
    "protocolFee": "50000",
    "networkFee": "200000",
    "solverFee": "100000"
  },
  "expiresAt": "2026-02-27T13:00:00Z",
  "quoteId": "quote_xyz789"
}
```

## Intents

### `POST /v1/intents`

Create a new swap intent (escrow). Returns an unsigned transaction for the user to sign.

**Request Body:**
```json
{
  "creator": "addr_test1qz...",
  "inputPolicyId": "",
  "inputAssetName": "",
  "inputAmount": "100000000",
  "outputPolicyId": "abc123def456",
  "outputAssetName": "544f4b454e",
  "minOutput": "49000000",
  "deadline": 1740700000,
  "partialFill": false,
  "senderAddress": "addr_test1qz...",
  "changeAddress": "addr_test1qz..."
}
```

**Response:**
```json
{
  "intentId": "clxyz789abc",
  "unsignedTx": "84a400818258...",
  "txHash": "abc123...",
  "status": "CREATED"
}
```

### `GET /v1/intents`

List intents with pagination and filters.

**Query Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `address` | string | Yes | Filter by creator address |
| `status` | string | No | Filter by status (CREATED, ACTIVE, FILLED, etc.) |
| `cursor` | string | No | Pagination cursor |
| `limit` | number | No | Max results per page (default: 20) |

**Response:**
```json
{
  "data": [
    {
      "intentId": "clxyz789abc",
      "status": "ACTIVE",
      "creator": "addr_test1qz...",
      "inputAsset": "lovelace",
      "inputAmount": "100000000",
      "outputAsset": "abc123.TOKEN",
      "minOutput": "49000000",
      "deadline": 1740700000,
      "createdAt": "2026-02-27T12:00:00Z"
    }
  ],
  "pagination": {
    "cursor": "clxyz789abc",
    "hasMore": true,
    "total": 42
  }
}
```

### `GET /v1/intents/:intentId`

Get detailed intent information.

**Response:**
```json
{
  "intentId": "clxyz789abc",
  "status": "FILLED",
  "creator": "addr_test1qz...",
  "inputAsset": "lovelace",

  "actualOutput": "49251234",

  "updatedAt": "2026-02-27T12:05:00Z"
```

Cancel an active intent. Returns an unsigned cancel transaction.

**Request Body:**
```json

}
```

## Transactions

### `POST /v1/tx/submit`

Submit a signed transaction (CBOR hex) to the Cardano network.

**Request Body:**
```json
{
  "signedTx": "84a400818258..."
}
```

**Response (200):**
```json
{
  "txHash": "abc123...",
  "status": "accepted"
}
```

**Response (400):**
```json
{
  "txHash": "",
  "status": "rejected",
  "error": "Transaction validation failed: ..."
}
```

### `POST /v1/tx/confirm`

Confirm a transaction on-chain and update related intent/order/pool status.

**Request Body:**
```json
{
  "txHash": "abc123...",
  "intentId": "intent_xyz",
  "orderId": null,
  "poolId": null,
  "action": "fill"
}
```

**Response:**
```json
{
  "status": "ok",
  "txHash": "abc123..."
}
```

### `GET /v1/tx/:txHash/status`

Check transaction confirmation status on-chain.

**Response:**
```json
{
  "txHash": "abc123...",
  "status": "confirmed",
  "confirmations": 12
}
```
