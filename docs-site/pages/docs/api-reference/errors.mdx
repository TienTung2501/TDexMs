# Error Codes

import { Callout } from 'nextra/components'

All API errors follow a consistent format with an HTTP status code and structured error body.

## Error Response Format

```json
{
  "status": "error",
  "code": "ERROR_CODE",
  "message": "Human-readable error description",
  "details": {}
}
```

---

## HTTP Status Mapping

| HTTP Status | Domain Error | Description |
|-------------|-------------|-------------|
| 400 | Various | Bad request / validation error |
| 401 | `UNAUTHORIZED` | Authentication required |
| 404 | `NOT_FOUND` | Resource not found |
| 429 | `RATE_LIMIT_EXCEEDED` | Too many requests |
| 502 | `EXTERNAL_SERVICE_ERROR` | Blockfrost / external service failure |
| 500 | `INTERNAL_ERROR` | Unexpected server error |

---

## Validation Errors

Validation errors use Zod schemas and return detailed field-level information.

```json
{
  "status": "error",
  "code": "VALIDATION_ERROR",
  "message": "Request validation failed",
  "details": {
    "fieldErrors": [
      {
        "field": "inputAmount",
        "message": "Expected string, received number",
        "code": "invalid_type"
      },
      {
        "field": "inputAsset",
        "message": "Required",
        "code": "invalid_type"
      }
    ]
  }
}
```

|------|------|-------------|

| `INTENT_CANCELLED` | 400 | Intent was already cancelled |

| `INTENT_CANCEL_FAILED` | 400 | Failed to build cancel transaction |

| Code | HTTP | Description |

## Error Handling Best Practices

<Callout type="info">
**Retry Strategy:** For `502` (external service) errors, implement exponential backoff with 3 retries.
For `429` (rate limit) errors, wait for the `Retry-After` header value before retrying.
</Callout>

```typescript
async function apiCall(url: string, options?: RequestInit) {
  const MAX_RETRIES = 3;

  for (let attempt = 0; attempt < MAX_RETRIES; attempt++) {
    const res = await fetch(url, options);

    if (res.ok) return res.json();

    if (res.status === 429) {
      const retryAfter = parseInt(res.headers.get('Retry-After') || '60');
      await new Promise(r => setTimeout(r, retryAfter * 1000));
      continue;
    }

    if (res.status === 502 && attempt < MAX_RETRIES - 1) {
      await new Promise(r => setTimeout(r, Math.pow(2, attempt) * 1000));
      continue;
    }

    const error = await res.json();
    throw new ApiError(error.code, error.message, res.status);
  }
}
```
