# Netting Engine

import { Callout } from 'nextra/components'

The netting engine offsets opposing swap flows to minimize AMM pool impact.

## Concept

When users swap in opposite directions (ADA→TOKEN and TOKEN→ADA), their flows can be **directly matched** without touching the pool:

```
Without netting:                    With netting:
┌─────┐  100 ADA   ┌──────┐       ┌─────┐          ┌─────┐
│ U1  │────────────▶│ Pool │       │ U1  │───80─────▶│ U2  │
└─────┘             │      │       └─────┘    ADA    └─────┘
┌─────┐  80 ADA    │      │              20 ADA
│ U2  │◀────────────│      │       ┌──────────▼──────────┐
└─────┘             └──────┘       │ Pool (residual only) │
                                   └─────────────────────┘
Total through pool: 180 ADA       Total through pool: 20 ADA
```

**Result**: 89% less pool impact → better execution for all users.

## Algorithm

```typescript
// NettingEngine.ts — static analyze()

static analyze(batch: Intent[], pool: PoolState): NettingPlan {
  // 1. Separate by direction
  const aToB = batch.filter(i => i.direction === 'AToB');
  const bToA = batch.filter(i => i.direction === 'BToA');

  // Single direction — no netting possible
  if (aToB.length === 0 || bToA.length === 0) {
    return this.singleDirectionPlan(batch, pool);
  }

  // 2. Calculate gross flows
  const grossA = aToB.reduce((sum, i) => sum + i.fillAmount, 0n);
  const grossB = bToA.reduce((sum, i) => sum + i.fillAmount, 0n);

  // 3. Convert to common denomination (asset A) via spot price
  //    spotPrice = reserveA / reserveB
  const grossBinA = (grossB * pool.reserveA) / pool.reserveB;

  // 4. Calculate net residual
  const netResidual = grossA > grossBinA
    ? grossA - grossBinA      // Net AToB flow
    : grossBinA - grossA;     // Net BToA flow

  const residualDirection = grossA > grossBinA ? 'AToB' : 'BToA';

  // 5. Only residual goes through AMM
  const ammOutput = AmmMath.calculateSwapOutput(
    netResidual,
    residualDirection === 'AToB' ? pool.reserveA : pool.reserveB,
    residualDirection === 'AToB' ? pool.reserveB : pool.reserveA,
    pool.feeNumerator
  );

  // 6. Allocate outputs proportionally
  const outputs = this.allocateOutputs(
    aToB, bToA, grossA, grossB, grossBinA, ammOutput, pool
  );

  const improvement = 1 - Number(netResidual) / Number(grossA + grossBinA);

  return { outputs, nettingImprovement: improvement, residualDirection };
}
```

## Detailed Numerical Example

### Setup

```
Pool:  reserveA = 100,000 ADA,  reserveB = 50,000 TOKEN
Price: 1 TOKEN = 2 ADA

Three intents:
  Intent 1: 10,000 ADA → TOKEN  (buy TOKEN)
  Intent 2: 15,000 ADA → TOKEN  (buy TOKEN)
  Intent 3:  6,000 TOKEN → ADA  (sell TOKEN)
```

### Step 1: Separate Directions

```
AToB (ADA → TOKEN): [Intent1: 10,000, Intent2: 15,000]
BToA (TOKEN → ADA): [Intent3: 6,000 TOKEN]
```

### Step 2: Gross Flows

```
grossA = 10,000 + 15,000 = 25,000 ADA
grossB = 6,000 TOKEN
```

### Step 3: Convert to Common Denomination

```
spotPrice = reserveA / reserveB = 100,000 / 50,000 = 2.0 ADA/TOKEN

grossBinA = 6,000 × 2.0 = 12,000 ADA equivalent
```

### Step 4: Net Residual

```
netResidual = |25,000 - 12,000| = 13,000 ADA
residualDirection = AToB (buy pressure dominates)

Netting offset: 12,000 ADA matched directly (48%)
Only 13,000 ADA goes through the AMM
```

### Step 5: AMM Swap (Residual Only)

```
AMM calculates: 13,000 ADA → TOKEN
fee = 13,000 × 30 / 10,000 = 39 ADA
inputAfterFee = 13,000 - 39 = 12,961
output = (50,000 × 12,961) / (100,000 + 12,961)
       = 648,050,000 / 112,961
       = 5,737 TOKEN (from AMM)
```

### Step 6: Allocate Outputs

**AToB intents** (buy TOKEN):
```
Total TOKEN available for buyers:
  From netting: 6,000 TOKEN (directly from Intent3)
  From AMM:     5,737 TOKEN
  Total:       11,737 TOKEN

Intent 1: 10,000/25,000 × 11,737 = 4,695 TOKEN
Intent 2: 15,000/25,000 × 11,737 = 7,042 TOKEN
```

**BToA intents** (sell TOKEN):
```
Intent 3 receives: 6,000 × 2.0 = 12,000 ADA
(matched directly from the ADA flow of Intent1 + Intent2)
```

### Improvement

```
Without netting: 25,000 + 12,000 = 37,000 ADA equivalent through pool
With netting:    13,000 ADA through pool

Netting improvement: 1 - 13,000/37,000 = 64.9%
```

<Callout type="info">
This 53–65% improvement is typical for mixed-direction batches. Single-direction batches (all buys or all sells) cannot be netted and go entirely through the AMM.
</Callout>

## Output Allocation

The `allocateOutputs()` function ensures fairness:

```typescript
static allocateOutputs(
  aToB: Intent[], bToA: Intent[],
  grossA: bigint, grossB: bigint,
  grossBinA: bigint, ammOutput: bigint,
  pool: PoolState
): OutputAllocation[] {
  const outputs: OutputAllocation[] = [];

  // For AToB intents: distribute total TOKEN proportionally
  const totalTokenForBuyers = grossB + ammOutput; // direct match + AMM
  for (const intent of aToB) {
    const share = (intent.fillAmount * totalTokenForBuyers) / grossA;
    outputs.push({
      address: intent.creatorAddress,
      asset: pool.assetB,
      amount: share,
    });
  }

  // For BToA intents: distribute ADA proportionally
  const totalAdaForSellers = grossBinA; // from netting offset
  for (const intent of bToA) {
    const intentInA = (intent.fillAmount * pool.reserveA) / pool.reserveB;
    const share = (intentInA * totalAdaForSellers) / grossBinA;
    outputs.push({
      address: intent.creatorAddress,
      asset: pool.assetA,
      amount: share,
    });
  }

  return outputs;
}
```

## Single-Direction Batches

When all intents go in the same direction, netting cannot help:

```typescript
static singleDirectionPlan(batch: Intent[], pool: PoolState): NettingPlan {
  const totalInput = batch.reduce((s, i) => s + i.fillAmount, 0n);

  const ammOutput = AmmMath.calculateSwapOutput(
    totalInput, pool.reserveIn, pool.reserveOut, pool.feeNumerator
  );

  // Allocate proportionally
  const outputs = batch.map(intent => ({
    address: intent.creatorAddress,
    asset: pool.outputAsset,
    amount: (intent.fillAmount * ammOutput) / totalInput,
  }));

  return { outputs, nettingImprovement: 0.0, residualDirection: batch[0].direction };
}
```

## Edge Cases

| Edge Case | Handling |
|-----------|----------|
| Single intent in batch | Direct AMM swap, no netting |
| All same direction | `singleDirectionPlan()` — 0% netting |
| Perfect offset (equal flows) | 100% netting — zero AMM usage |
| Rounding remainder | Last intent in list absorbs rounding dust |
| Empty AMM output | Batch skipped if AMM returns 0 |
| MinOutput validation | Each intent's proportional output checked ≥ proportional minOutput |
