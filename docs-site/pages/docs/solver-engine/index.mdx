## 6-Stage Pipeline

| Stage | Module | Purpose |
|-------|--------|---------|
| 1 | `IntentCollector` | Query Blockfrost for escrow UTxOs, parse datums |
| 2 | `SolverEngine.filter` | Remove stale, expired, and in-progress intents |
| 3 | `RouteOptimizer` | Find best route (direct / multi-hop / partial fill) |
| 4 | `BatchBuilder` | Group by pool, enforce TX limits, split batches |
| 5 | `NettingEngine` | Offset opposing flows, calculate net AMM residual |
| 6 | `TxSubmitter` | Build settlement TX, sign, submit, confirm |

## Netting Advantage

Netting can dramatically reduce AMM pool impact:

```
Without netting:                     With netting:
  Intent 1: 100 ADA вҶ’ TOKEN         Net flow: 20 ADA вҶ’ TOKEN
  Intent 2:  80 ADA вҶҗ TOKEN         (80 ADA offset directly)
  Total AMM: 180 ADA through pool   Total AMM: 20 ADA through pool
  Price impact: HIGH                 Price impact: LOW
```

In a batch of 3 intents, netting achieved **53% improvement** вҖ” only 47% of the total flow hit the AMM.

## Configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `SOLVER_ENABLED` | `true` | Enable/disable solver |
| `SOLVER_INTERVAL` | `30000` | Milliseconds between iterations |
| `SOLVER_MAX_BATCH_SIZE` | `15` | Maximum intents per TX |
| `SOLVER_MIN_PROFIT_LOVELACE` | `0` | Minimum profit threshold |
| `SOLVER_PARTIAL_FILL_ENABLED` | `true` | Allow partial fills |
| `SOLVER_MAX_TX_CPU` | `14000000000` | 14B CPU execution units |
| `SOLVER_MAX_TX_MEM` | `10000000` | 10M memory units |
