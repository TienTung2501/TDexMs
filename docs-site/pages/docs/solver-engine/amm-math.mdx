# AMM Math

All mathematical functions used by the solver and on-chain validators. Implementations use **BigInt integer arithmetic** exclusively — no floating point.

**Source**: `backend/src/solver/AmmMath.ts` (236 lines)
**On-chain**: `lib/solvernet/math.ak` (328 lines, 10 unit tests)

## Swap Calculation

### calculateSwapOutput

Given an input amount, calculate the output after fees:

$$output = \frac{reserveOut \times inputAfterFee}{reserveIn + inputAfterFee}$$

Where:

$$inputAfterFee = inputAmount - fee$$
$$fee = \frac{inputAmount \times feeNumerator}{feeDenominator}$$

```typescript
static calculateSwapOutput(
  inputAmount: bigint,
  reserveIn: bigint,
  reserveOut: bigint,
  feeNumerator: bigint
): bigint {
  const FEE_DENOMINATOR = 10000n;
  const fee = (inputAmount * feeNumerator) / FEE_DENOMINATOR;
  const inputAfterFee = inputAmount - fee;
  const numerator = reserveOut * inputAfterFee;
  const denominator = reserveIn + inputAfterFee;
  return numerator / denominator;  // Integer division (floor)
}
```

**Example**:
```
Input:     100,000 ADA
ReserveIn: 1,000,000 ADA
ReserveOut: 500,000 TOKEN
Fee:       30 (0.30%)

fee = 100,000 × 30 / 10,000 = 300 ADA
inputAfterFee = 99,700 ADA
output = 500,000 × 99,700 / (1,000,000 + 99,700)
       = 49,850,000,000 / 1,099,700
       = 45,330 TOKEN
```

### calculateProtocolFee

Protocol's share of the trading fee:

$$protocolFee = \frac{fee}{protocolFeeShare}$$

```typescript
static calculateProtocolFee(
  fee: bigint,
  protocolFeeShare: bigint = 6n  // 1/6 of fee
): bigint {
  return fee / protocolFeeShare;
}
```

### calculateRequiredInput (Reverse)

Given a desired output, calculate the required input:

$$requiredInput = \frac{reserveIn \times output \times feeDenominator}{(reserveOut - output) \times (feeDenominator - feeNumerator)}$$

```typescript
static calculateRequiredInput(
  outputAmount: bigint,
  reserveIn: bigint,
  reserveOut: bigint,
  feeNumerator: bigint
): bigint {
  const FEE_DENOMINATOR = 10000n;
  const numerator = reserveIn * outputAmount * FEE_DENOMINATOR;
  const denominator = (reserveOut - outputAmount) * (FEE_DENOMINATOR - feeNumerator);
  return numerator / denominator + 1n;  // Round up
}
```

## Price Functions

### meetsLimitPrice

Check if a swap execution meets a limit order's price target:

$$\frac{outputAmount}{inputAmount} \geq \frac{minOutput}{budget}$$

In integer arithmetic (cross multiply to avoid division):

```typescript
static meetsLimitPrice(
  outputAmount: bigint,
  inputAmount: bigint,
  minOutput: bigint,
  budget: bigint
): boolean {
  return outputAmount * budget >= minOutput * inputAmount;
}
```

### Price Impact

$$impact = 1 - \frac{effectivePrice}{spotPrice}$$

Where:
$$spotPrice = \frac{reserveOut}{reserveIn}$$
$$effectivePrice = \frac{output}{input}$$

## Liquidity Calculations

### calculateInitialLp

First liquidity deposit:

$$LP_{initial} = \sqrt{amountA \times amountB} - minimumLiquidity$$

```typescript
static calculateInitialLp(amountA: bigint, amountB: bigint): bigint {
  const MINIMUM_LIQUIDITY = 1000n;
  return bigIntSqrt(amountA * amountB) - MINIMUM_LIQUIDITY;
}
```

`minimumLiquidity` (1000 tokens) is permanently locked to prevent a zero-liquidity griefing attack.

### calculateDepositLp

Subsequent deposits:

$$LP_{minted} = \min\left(\frac{amountA \times totalLP}{reserveA}, \frac{amountB \times totalLP}{reserveB}\right)$$

```typescript
static calculateDepositLp(
  amountA: bigint, amountB: bigint,
  reserveA: bigint, reserveB: bigint,
  totalLp: bigint
): bigint {
  const lpFromA = (amountA * totalLp) / reserveA;
  const lpFromB = (amountB * totalLp) / reserveB;
  return lpFromA < lpFromB ? lpFromA : lpFromB;
}
```

### calculateWithdrawal

LP token redemption:

$$outputA = \frac{lpBurned \times reserveA}{totalLP}$$
$$outputB = \frac{lpBurned \times reserveB}{totalLP}$$

```typescript
static calculateWithdrawal(
  lpBurned: bigint,
  reserveA: bigint, reserveB: bigint,
  totalLp: bigint
): { amountA: bigint; amountB: bigint } {
  return {
    amountA: (lpBurned * reserveA) / totalLp,
    amountB: (lpBurned * reserveB) / totalLp,
  };
}
```

## Root K (Manipulation Resistance)

The pool validator tracks $\sqrt{k}$ to prevent reserves manipulation:

$$rootK = \sqrt{reserveA \times reserveB}$$

After every swap, the validator checks:

$$rootK_{new} \geq rootK_{old}$$

```typescript
static calculateRootK(reserveA: bigint, reserveB: bigint): bigint {
  return bigIntSqrt(reserveA * reserveB);
}
```

### bigIntSqrt (Newton's Method)

```typescript
static bigIntSqrt(n: bigint): bigint {
  if (n < 0n) throw new Error('Square root of negative');
  if (n < 2n) return n;

  let x = n;
  let y = (x + 1n) / 2n;
  while (y < x) {
    x = y;
    y = (x + n / x) / 2n;
  }
  return x;
}
```

## Partial Fill Calculations

### calculateMaxAbsorbableAmount

Binary search for the maximum input that keeps output/input ratio above the limit price:

```typescript
static calculateMaxAbsorbableAmount(
  reserveIn: bigint, reserveOut: bigint,
  feeNumerator: bigint,
  minOutput: bigint, budget: bigint
): bigint {
  let lo = 0n;
  let hi = reserveIn;  // Upper bound

  while (lo < hi) {
    const mid = (lo + hi + 1n) / 2n;
    const output = this.calculateSwapOutput(mid, reserveIn, reserveOut, feeNumerator);

    if (this.meetsLimitPrice(output, mid, minOutput, budget)) {
      lo = mid;
    } else {
      hi = mid - 1n;
    }
  }
  return lo;
}
```

### calculateMaxIntentFill

Binary search for the maximum fill amount that satisfies the intent's minOutput:

```typescript
static calculateMaxIntentFill(
  intent: Intent, pool: PoolState
): bigint {
  const proportionalMinOutput = (fillAmount: bigint) =>
    (fillAmount * intent.minOutput) / intent.inputAmount;

  let lo = 0n;
  let hi = intent.remainingInput;

  while (lo < hi) {
    const mid = (lo + hi + 1n) / 2n;
    const output = this.calculateSwapOutput(
      mid, pool.reserveIn, pool.reserveOut, pool.feeNumerator
    );

    if (output >= proportionalMinOutput(mid)) {
      lo = mid;
    } else {
      hi = mid - 1n;
    }
  }
  return lo;
}
```

## Numerical Safety

<Callout type="warning">
All operations use BigInt to prevent overflow and precision loss. Key rules:

- **Division**: Always integer floor division (`/`). Round up explicitly with `+ 1n` when needed.
- **Ordering**: Multiply before dividing to preserve precision.
- **Overflow**: BigInt has no overflow — arbitrary precision.
- **Equivalence**: On-chain Aiken and off-chain TypeScript functions must produce identical results.
</Callout>
