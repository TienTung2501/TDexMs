# Components

import { Callout } from 'nextra/components'

Key UI components in the SolverNet DEX frontend.

## Component Architecture

```
Layout Components
├── Header          # Navigation, wallet connect, network badge
├── Footer          # Links, credits
└── Sidebar         # Mobile navigation

Trading Components
├── SwapCard        # Main swap interface
│   ├── TokenSelect # Token picker with search
│   ├── AmountInput # Token amount input with balance
│   └── SwapButton  # Sign & submit swap
├── PriceChart      # TradingView candlestick chart
├── RecentTrades    # Live trade feed from pool
└── QuoteDisplay    # Price impact, fees, route info

Pool Components
├── PoolTable       # Pool list with sorting
├── PoolCard        # Individual pool summary
├── LiquidityForm   # Add/remove liquidity
├── PoolChart       # TVL & volume history
└── PoolSwaps       # Recent swaps for a pool

Order Components
├── OrderEntry      # Create limit/DCA/stop-loss orders
├── OrderList       # Active orders with cancel
└── OrderHistory    # Completed/cancelled orders

Portfolio Components
├── PortfolioSummary    # Total balance, allocation chart
├── OpenPositions       # Active intents & orders
├── TradeHistory        # Completed trades
├── LPPositions         # Liquidity provider positions
└── PerformanceChart    # Portfolio value over time

Admin Components
├── AdminDashboard      # Metrics, TVL, volume cards
├── ProtocolDeploy      # Deploy settings/factory
├── SettingsPanel       # Update protocol settings
├── FeeCollector        # Collect protocol fees
├── SolverControl       # Start/stop/trigger solver
└── PoolManager         # Admin pool operations
```

## Key Components

### SwapCard

The primary trading interface. Handles the full swap flow:

1. **Token Selection** — User picks input/output tokens from dynamic registry
2. **Amount Input** — Enter amount with real-time quote fetching
3. **Quote Display** — Shows output amount, price impact, fees, route
4. **Swap Execution** — Builds escrow TX → wallet signs → submit to chain

```tsx
<SwapCard>
  <TokenInput
    label="You Pay"
    token={inputToken}
    amount={inputAmount}
    balance={walletBalance}
    onTokenChange={handleTokenChange}
    onAmountChange={handleAmountChange}
  />
  <SwapDirectionButton onClick={handleReverse} />
  <TokenInput
    label="You Receive"
    token={outputToken}
    amount={quote?.outputAmount}
    readOnly
  />
  <QuoteDetails
    priceImpact={quote?.priceImpact}
    minimumOutput={quote?.minOutput}
    fees={quote?.estimatedFees}
    route={quote?.route}
  />
  <SwapButton
    disabled={!quote || !connected}
    loading={isSwapping}
    onClick={handleSwap}
  />
</SwapCard>
```

### TokenSelect

Token picker with search, filtering, and Uniswap-style UX:

- Fetches available tokens from `GET /v1/tokens`
- Groups tokens into "Popular" and "All Tokens"
- Search by ticker, name, or policy ID
- Shows wallet balance for each token
- Prevents selecting the same token for both input and output

### PriceChart

TradingView-style candlestick chart with:

- Multiple timeframes (1m, 5m, 15m, 1h, 4h, 1D, 1W)
- OHLCV data from `GET /v1/chart/candles`
- Real-time price updates via WebSocket
- Volume bars overlay
- Responsive sizing

## UI Components (shadcn/ui)

The frontend uses [shadcn/ui](https://ui.shadcn.com) for accessible, customizable UI primitives:

| Component | Usage |
|-----------|-------|
| `Button` | All action buttons with variants |
| `Card` | Content containers and widgets |
| `Dialog` | Token select modal, confirmations |
| `Input` | Amount inputs, search fields |
| `Select` | Dropdowns for timeframes, filters |
| `Table` | Pool lists, trade history |
| `Tabs` | Section switching (Swap/Limit/DCA) |
| `Badge` | Status indicators, network labels |
| `Skeleton` | Loading states |
| `Toast` | Success/error notifications |

## Theming

Dark mode is the default theme, managed via `next-themes`:

```tsx
// providers/theme-provider.tsx
<ThemeProvider attribute="class" defaultTheme="dark">
  {children}
</ThemeProvider>
```

Custom theme colors extend Tailwind's palette:

```css
:root {
  --primary: 210 100% 50%;     /* Blue */
  --background: 222 47% 11%;   /* Dark navy */
  --card: 223 47% 14%;         /* Slightly lighter */
  --accent: 142 76% 36%;       /* Green for positive values */
  --destructive: 0 84% 60%;    /* Red for negative values */
}
```
