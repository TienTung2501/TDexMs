# Production Deployment

import { Callout, Steps, Tabs } from 'nextra/components'

Step-by-step guides for deploying SolverNet DEX to production services.

## Backend: Render

<Steps>

### Create a Render Account

Go to [render.com](https://render.com) and sign up.

### Create Web Service

1. Click **New → Web Service**
2. Connect your GitHub repository
3. Configure:
   - **Name:** `solvernet-backend`
   - **Root Directory:** (leave empty — uses monorepo root)
   - **Runtime:** Node
   - **Build Command:** `pnpm install && cd backend && pnpm exec prisma generate`
   - **Start Command:** `cd backend && pnpm exec tsx src/index.ts`
   - **Instance Type:** Starter ($7/mo) or higher

### Set Environment Variables

Add all required variables in the Render dashboard under **Environment**:

```
NODE_ENV=production
PORT=3001
DATABASE_URL=postgresql://...
BLOCKFROST_PROJECT_ID=...
CARDANO_NETWORK=preprod
CORS_ORIGIN=https://your-frontend.vercel.app
JWT_SECRET=your-secure-secret
SOLVER_ENABLED=true
SOLVER_SEED_PHRASE=...
```

### Deploy

Render auto-deploys on push to your main branch. Monitor the deploy logs for any issues.

### Verify

```bash
curl https://solvernet-backend.onrender.com/v1/health
```

</Steps>

<Callout type="info">
Render's free tier spins down after inactivity. Use a paid plan for the solver engine to run continuously.
</Callout>

## Frontend: Vercel

<Steps>

### Create Vercel Project

1. Go to [vercel.com](https://vercel.com)
2. Import your GitHub repository
3. Configure:
   - **Framework Preset:** Next.js
   - **Root Directory:** `frontend`
   - **Build Command:** (auto-detected)
   - **Output Directory:** (auto-detected)

### Set Environment Variables

```
NEXT_PUBLIC_API_URL=https://solvernet-backend.onrender.com
NEXT_PUBLIC_WS_URL=wss://solvernet-backend.onrender.com
NEXT_PUBLIC_CARDANO_NETWORK=preprod
```

### Configure `vercel.json`

The frontend `vercel.json` handles routing:

```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/" }
  ]
}
```

### Deploy

Push to your main branch. Vercel auto-deploys and provides a production URL.

</Steps>

## Database: Supabase

<Steps>

### Create Project

1. Go to [supabase.com](https://supabase.com)
2. Create a new project
3. Choose a region close to your backend server

### Get Connection String

Navigate to **Settings → Database → Connection string (URI)** and copy.

### Apply Schema

```bash
DATABASE_URL="postgresql://..." pnpm exec prisma db push
```

### Enable Connection Pooling

For production, enable Supabase's built-in PgBouncer:
- Go to **Settings → Database → Connection Pooling**
- Use the pooled connection string for your backend

</Steps>

## Cache: Upstash Redis

<Steps>

### Create Database

1. Go to [upstash.com](https://upstash.com)
2. Create a new Redis database
3. Choose the same region as your backend

### Get Credentials

Copy the REST URL and token from the dashboard.

```env
UPSTASH_REDIS_URL=https://xxxxx.upstash.io
UPSTASH_REDIS_TOKEN=AXxxxx...
```

</Steps>

## Monitoring

### Health Checks

Set up uptime monitoring on:
- `GET /v1/health/ready` — Returns 200 when system is operational

### Recommended Tools

| Tool | Purpose |
|------|---------|
| **UptimeRobot** | Uptime monitoring (ping `/v1/health/ready` every 5 min) |
| **Render Logs** | Backend application logs |
| **Supabase Dashboard** | Database metrics and slow queries |
| **Blockfrost Dashboard** | API usage and remaining quota |

### Key Metrics to Monitor

| Metric | Alert Threshold |
|--------|----------------|
| API response time | > 2s average |
| Error rate | > 1% of requests |
| Database connections | > 80% pool utilization |
| Blockfrost usage | > 80% daily quota |
| Solver success rate | < 90% batch success |
| WebSocket connections | > 1000 concurrent |

## SSL / Domain Setup

### Custom Domain (Vercel)

1. Go to your Vercel project → **Settings → Domains**
2. Add your custom domain (e.g., `app.solvernet.io`)
3. Configure DNS: Add a CNAME record pointing to Vercel

### Custom Domain (Render)

1. Go to your Render service → **Settings → Custom Domains**
2. Add your domain (e.g., `api.solvernet.io`)
3. Configure DNS as shown in the Render dashboard

<Callout type="warning">
Update `CORS_ORIGIN` in the backend to match your production frontend domain after setting up custom domains.
</Callout>
