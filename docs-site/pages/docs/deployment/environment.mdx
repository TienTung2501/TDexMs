# Environment Configuration

import { Callout } from 'nextra/components'

Production environment setup for each deployment target.

<Callout type="warning">
See [Configuration](/docs/getting-started/configuration) for the complete variable reference. This page covers production-specific settings.
</Callout>

## Production Essentials

These variables **must** be changed from development defaults for production:

| Variable | Why |
|----------|-----|
| `NODE_ENV` | Set to `production` (enables optimizations, reduces logging) |
| `JWT_SECRET` | Use a cryptographically random string (min 32 chars) |
| `CORS_ORIGIN` | Set to your production domain(s) |
| `RATE_LIMIT_MAX` | Lower for production (e.g., 50) |
| `SOLVER_SEED_PHRASE` | Use a dedicated funded wallet |
| `DATABASE_URL` | Use SSL-enabled connection string |

## Network Configuration

### Preprod Testnet

```env
CARDANO_NETWORK=preprod
BLOCKFROST_URL=https://cardano-preprod.blockfrost.io/api/v0
BLOCKFROST_PROJECT_ID=preprodXXXXXXXXXXXXX
```

### Mainnet

```env
CARDANO_NETWORK=mainnet
BLOCKFROST_URL=https://cardano-mainnet.blockfrost.io/api/v0
BLOCKFROST_PROJECT_ID=mainnetXXXXXXXXXXXXX
```

<Callout type="error">
**Mainnet Warning:** Mainnet operations use real ADA. Thoroughly test on Preprod before deploying to Mainnet.
</Callout>

## Security Hardening

### JWT Secret

```bash
# Generate a secure random secret
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
```

### CORS

```env
# Single origin
CORS_ORIGIN=https://app.solvernet.io

# Multiple origins
CORS_ORIGIN=https://app.solvernet.io,https://admin.solvernet.io
```

### Rate Limiting

```env
# Tighter limits for production
RATE_LIMIT_WINDOW_MS=60000
RATE_LIMIT_MAX=50
```

## Solver Configuration

```env
SOLVER_ENABLED=true
SOLVER_SEED_PHRASE="word1 word2 ... word24"
SOLVER_ADDRESS=addr1q...
SOLVER_BATCH_WINDOW_MS=10000     # 10s batch window (production)
SOLVER_MAX_RETRIES=5
SOLVER_MIN_PROFIT_LOVELACE=500000 # Higher threshold for mainnet

ORDER_ROUTES_ENABLED=true
ORDER_EXECUTOR_ENABLED=true
```

<Callout type="error">
**Never expose** `SOLVER_SEED_PHRASE` in logs, version control, or client-side code. Use your hosting platform's secret management.
</Callout>

## Database SSL

For Supabase and other cloud PostgreSQL providers, the connection string should include SSL:

```env
DATABASE_URL="postgresql://user:pass@host:5432/db?sslmode=require"
```

## Logging in Production

```env
# Production: only log warnings and errors
LOG_LEVEL=warn

# For debugging production issues:
LOG_LEVEL=info
```
