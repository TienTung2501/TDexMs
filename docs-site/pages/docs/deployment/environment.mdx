# Environment Variables

import { Callout, Tabs } from 'nextra/components'

## Overview

SolverNet DEX uses environment variables for all sensitive and environment-specific configuration. Never commit `.env` files to version control.

## Production Environment Reference

### Required Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `DATABASE_URL` | PostgreSQL connection string | `postgresql://user:pass@host:5432/db?sslmode=require` |
| `SOLVER_PRIVATE_KEY` | Solver wallet signing key (Ed25519 hex) | `5820...` |
| `BLOCKFROST_API_KEY` | Blockfrost project API key | `preprodXXXXX` |

### Network Configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `CARDANO_NETWORK` | `Preprod` | `Mainnet` \| `Preprod` \| `Preview` |
| `BLOCKFROST_URL` | `https://cardano-preprod.blockfrost.io/api` | Blockfrost base URL |

### Server Configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `PORT` | `3001` | HTTP server port |
| `CORS_ORIGIN` | `http://localhost:3000` | Allowed CORS origins (comma-separated) |
| `LOG_LEVEL` | `info` | `debug` \| `info` \| `warn` \| `error` |

### Solver Configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `SOLVER_INTERVAL_MS` | `15000` | Solve cycle interval in milliseconds |
| `MIN_BATCH_SIZE` | `2` | Minimum intents per batch |
| `MAX_BATCH_SIZE` | `30` | Maximum intents per batch |
| `MATCH_TOLERANCE_PERCENT` | `2` | Netting match tolerance (%) |
| `PROTOCOL_FEE_PERCENT` | `0.3` | Protocol fee (%) |

### Smart Contract Addresses

| Variable | Description |
|----------|-------------|
| `FACTORY_VALIDATOR_ADDRESS` | Factory validator script address |
| `POOL_VALIDATOR_ADDRESS` | Pool validator script address |

<Callout type="warning">
**Security Checklist:**
- Never expose `SOLVER_PRIVATE_KEY` in logs or API responses
- Use `sslmode=require` for production database connections
- Set `CORS_ORIGIN` to your exact frontend domain (no wildcards in production)
- Rotate `BLOCKFROST_API_KEY` regularly
- Use secret management (e.g., Render secrets, Vercel env, AWS Secrets Manager)
</Callout>
