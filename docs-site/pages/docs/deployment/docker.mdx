## Dockerfile

The project includes a multi-stage Dockerfile for the backend:

```dockerfile
# Stage 1: Build
FROM node:20-alpine AS builder
WORKDIR /app

# Install pnpm
RUN npm install -g pnpm

# Copy workspace config
COPY package.json pnpm-workspace.yaml pnpm-lock.yaml ./
COPY backend/package.json backend/
COPY patches/ patches/

# Install dependencies
RUN pnpm install --frozen-lockfile

# Copy source
COPY backend/ backend/
COPY tsconfig.base.json ./

# Generate Prisma client
RUN cd backend && pnpm exec prisma generate

# Stage 2: Runtime
FROM node:20-alpine
WORKDIR /app

RUN npm install -g pnpm

COPY --from=builder /app/ /app/

EXPOSE 3001
CMD ["pnpm", "--filter", "backend", "exec", "tsx", "src/index.ts"]
```

## Docker Compose (Development)

```yaml
# docker-compose.dev.yml
version: '3.8'
services:
  postgres:
    image: postgres:15-alpine
    environment:
      POSTGRES_DB: solvernet
      POSTGRES_USER: solvernet
      POSTGRES_PASSWORD: solvernet
    ports:
      - "5432:5432"
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U solvernet"]
      interval: 5s
      timeout: 5s
      retries: 5

  backend:
    build: .
    ports:
      - "3001:3001"
    environment:
      - DATABASE_URL=postgresql://solvernet:solvernet@postgres:5432/solvernet
      - NODE_ENV=development
      - PORT=3001
    depends_on:
      postgres:
        condition: service_healthy
    volumes:
      - ./backend/src:/app/backend/src

volumes:
  pgdata:
```

## Docker Compose (Production)

```yaml
# docker-compose.yml
version: '3.8'
services:
  backend:
    build: .
    restart: always
    ports:
      - "3001:3001"
    environment:
      - NODE_ENV=production
      - PORT=3001
      - DATABASE_URL=${DATABASE_URL}
      - BLOCKFROST_PROJECT_ID=${BLOCKFROST_PROJECT_ID}
      - BLOCKFROST_URL=${BLOCKFROST_URL}
      - CARDANO_NETWORK=${CARDANO_NETWORK}
      - UPSTASH_REDIS_URL=${UPSTASH_REDIS_URL}
      - UPSTASH_REDIS_TOKEN=${UPSTASH_REDIS_TOKEN}
      - SOLVER_ENABLED=${SOLVER_ENABLED}
      - SOLVER_SEED_PHRASE=${SOLVER_SEED_PHRASE}
      - CORS_ORIGIN=${CORS_ORIGIN}
      - JWT_SECRET=${JWT_SECRET}
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3001/v1/health/ready"]
      interval: 30s
      timeout: 10s
      retries: 3
```

## Commands

```bash
# Development
docker compose -f docker-compose.dev.yml up -d

# Production
docker compose up -d --build

# View logs
docker compose logs -f backend

# Scale (if using Docker Swarm)
docker service scale solvernet_backend=3

# Push schema to database
docker compose exec backend pnpm exec prisma db push
```

## Health Checks

The backend exposes two health endpoints:

- `GET /v1/health` — Full system status (database, Blockfrost, cache)
- `GET /v1/health/ready` — Simple readiness probe (returns 200 or 503)

Use `/v1/health/ready` for Docker health checks and Kubernetes readiness probes.
