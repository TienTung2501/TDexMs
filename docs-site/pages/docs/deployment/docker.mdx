## Production Dockerfile

The backend uses a multi-stage Docker build for a lean production image:

```dockerfile
# Stage 1: Build
FROM node:20-alpine AS builder
RUN corepack enable && corepack prepare pnpm@9.15.0 --activate

WORKDIR /app
COPY package.json pnpm-lock.yaml pnpm-workspace.yaml ./
COPY backend/package.json backend/
RUN pnpm install --frozen-lockfile

COPY backend/ backend/
COPY tsconfig.base.json .
RUN cd backend && pnpm exec prisma generate
RUN cd backend && pnpm exec tsc --project tsconfig.json
RUN cd backend && pnpm deploy --filter=backend --prod /prod/backend

# Stage 2: Run
FROM node:20-alpine AS runner
RUN addgroup --system --gid 1001 solvernet
RUN adduser --system --uid 1001 solvernet

WORKDIR /app
COPY --from=builder /prod/backend/node_modules ./node_modules
COPY --from=builder /app/backend/dist ./dist
COPY --from=builder /app/backend/prisma ./prisma

USER solvernet
EXPOSE 3001

CMD ["sh", "-c", "npx prisma migrate deploy && node dist/index.js"]
```

**Image properties:**
- Base: `node:20-alpine` (~50 MB)
- Non-root user: `solvernet` (UID 1001)
- Production dependencies only (`pnpm deploy --prod`)
- Runs migrations on startup

## Development Docker Compose

For local development with a full Cardano stack:

```yaml
# docker-compose.dev.yml
version: '3.8'
services:
  postgres:
    image: postgres:16-alpine
    ports:
      - "5432:5432"
    environment:
      POSTGRES_USER: solvernet
      POSTGRES_PASSWORD: solvernet
      POSTGRES_DB: solvernet
    volumes:
      - pgdata:/var/lib/postgresql/data

  cardano-node:
    image: ghcr.io/intersectmbo/cardano-node:10.4.1
    command: run --config /config/config.json --topology /config/topology.json
    volumes:
      - cardano-data:/data
      - ./config/cardano:/config
    # Initial sync: 4-8 hours for preprod

  ogmios:
    image: cardanosolutions/ogmios:v6.11.2
    ports:
      - "1337:1337"
    depends_on:
      - cardano-node

  kupo:
    image: cardanosolutions/kupo:v2.9.0
    ports:
      - "1442:1442"
    depends_on:
      - cardano-node

volumes:
  pgdata:
  cardano-data:
```

<Callout type="info">
The development docker-compose includes a full Cardano node, Ogmios, and Kupo. For most development, you can skip these and use Blockfrost directly — it's much faster to set up.
</Callout>

## Building & Running

```bash
# Build the Docker image
docker build -t decentralize-dex .

# Run with environment file
docker run -d \
  --name solvernet \
  --env-file backend/.env \
  -p 3001:3001 \
  decentralize-dex

# View logs
docker logs -f solvernet

# Stop
docker stop solvernet
```

## Docker Compose for Production

```yaml
version: '3.8'
services:
  backend:
    build: .
    ports:
      - "3001:3001"
    env_file:
      - backend/.env
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3001/v1/health/ready"]
      interval: 30s
      timeout: 10s
      retries: 3
```
